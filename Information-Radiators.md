## Task-9 : Team goals and objectives

- Assigned To :

Authentication module: "This module should handle the admin login, signup, and profile edit. The new admin should have an option to sign up to the system, to log in to the system, and should be able to view their profile and update/make changes to the profile as required. If they forgot the password they should have an option to retrieve it."
Email module: "This module should handle the email templates and the automatic email triggering. The admin should be able to trigger the automatic thank you email to all the donors whose inventory details are entered into the system. There should be an option provided to donors for email opt-in. If they do opt-in then the material has to be assigned to a program (Youth program bicycle, Retail materials, Earn-a-bike) 
1. If it is a youth program bicycle then
  a. After the program assignment, a second email has to be triggered with a photo of the bicycle being cleaned or refurbished. 
  b. After 2 weeks from the above email, an email with a picture of the class doing safety drills has to be sent
  c. After 2 weeks from the above email, an email with a photo of the class/graduating student/bike has to be sent.
2. If it is a retail sales bike donation then
  a. To the donor(s) who has donated the bicycle
    i. After the program assignment, a second email has to be triggered with a photo of the bicycle being cleaned or refurbished. 
    ii. After the sale, an email with the amount raised from the sale, and the funds used for should be sent
  b. To the donor(s) who has purchased the bicycle
   i. After 3 to 4 weeks of purchase, an email with opt-in link and thanks for the purchase note etc has to be sent"
Admin - Checkin donors and donations module: "This module should handle the registration of donors and the donation. The admin should be able to register the new donor(s) in the system either by directly entering their details in the application or by uploading an Excel with the details of donor(s) in a specific template. Also, the admin should be able to register the material donation details in the system once the registration of the donor is done. They should be able to search for a donor(s) update their details or delete them and add a new donation/update the donation details. No changes to the donation can be made once it gets assigned to a program."
Admin- Program checkin for opted-in users module: "This module should handle the program assignment of the materials and image uploading. The admin should be able to view the list of donations along with details of the donor and the option to assign it to a program (single or bulk update). They should be able to search with a material name and also bulk assign them to a program. They should also be able to search with the donor's name and assign their donation(s) to a program. The admin should be able to filter with the program and upload the group image. They should also be able to search for a donor or a donation and upload the image specific to that donor or donation."

## Task-13: Incident and bug tracking

- Assigned To :
- Explain :

This part aids the team in monitoring issues and defects found during development or brought to their attention by users.
Execution:
Track issues and defects with GitHub Issues. Each issue is represented by a particular defect or occurrence, enabling team members to work together to find, rank, and fix them.
Show a list of open issues pertaining to bugs and occurrences. Important details including the issue title, assignee, severity, status (open or closed), and creation/update timestamps should be included in this list.
Label problems according to their seriousness (critical, major, minor, etc.) or any other pertinent factor. This aids in setting priorities and efficiently directing the resolution process.
Provide filtering tools so that team members can see issues according to labels, assignee, or severity, among other variables.

For each issue, including links to pertinent code files or changes to aid in troubleshooting and resolution.
In order to reflect changes in the state of issues or newly produced issues, updating the incident and bug tracking component in real-time.
Personalization and Interaction:
Permitting team members to use the information radiator interface to directly generate new issues. This makes it possible to report issues or defects as soon as they are found.
Enable team members to leave comments and engage in dialogue on any issue within the information radiator. This will enable team membets to ask questions, exchange insights, and offer updates.
Make it possible for team members to subscribe to notifications for certain issues or groups of issues so they are always aware of critical updates.

Integrating project management software and chat services (like Slack) to promote easy communication and teamwork when resolving issues.
The team can efficiently monitor, manage, and fix issues and bugs by incorporating GitHub's issue tracking feature into the information radiator. This ensures the stability and quality of the program being created.
